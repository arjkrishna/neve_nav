0) What the code currently does (important correction)
Current ‚Äúshortest path‚ÄĚ for reward

The reward term you‚Äôre referring to is PathLengthDelta (eve/eve/reward/pathlengthdelta.py).

It uses pathfinder.path_length, which comes from BruteForceBFS (eve/eve/pathfinder/bruteforcebfs.py).

Key detail: despite the name, this pathfinder is not Dijkstra right now. It builds edge lengths, but the path search itself is unweighted BFS and it uses the first BFS path it finds (fewest branching nodes), not necessarily the smallest-length route.

So if you want ‚Äúmain centerline route to target‚ÄĚ in a principled way, you‚Äôll likely want to implement weighted shortest path (Dijkstra) on that same branching-point graph.

1) What you want to add to the state

You want:

Where the centerlines are available

For DualDeviceNav, the centerlines come from JSON and become branches:

eve_bench/eve_bench/dualdevicenav.py ‚Üí load_branches(...) returns a list of BranchWithRadii

2) Defining ‚Äúdesired path‚ÄĚ and which centerline points are on it
Graph you already have (branching-point connectivity)

BruteForceBFS builds:

nodes = branching points (BranchingPoint)

edges exist between branching points that share a ‚Äúconnection‚ÄĚ branch

each edge has:

points: the polyline segment along the branch between those branching points

length: arc length of that segment

That‚Äôs created in:

BruteForceBFS._initialize_node_connections(...)

What‚Äôs missing for your goal

For your mask ‚Äúwhich branch is on the path‚ÄĚ, you want the shortest path as a sequence of branches (not just nodes).

Right now BPConnection stores (length, points) but not which branch produced it.
To get a clean formulation, I‚Äôd modify it to store the branch too:

and in _initialize_node_connections set branch=connection.

Then after you compute the path nodes, you can recover:

mask per branch m_j = 1[branch_j ‚ąą B_path]

mask per centerline point M_{j,k} = m_j (or finer if you want partial-branch masks)
